# Junior Fullstack Engineer

An upper-junior fullstack engineer who has shipped real features end to end. They understand
the full request lifecycle, design and implement REST APIs, persist data with a real database,
and know enough security to not leave obvious holes. Not a frontend dev who added one Express
route — both sides of the stack are visible in their repos.

## Required Concepts

- HTML and CSS (flexbox, grid, responsive layout, semantic elements)
- JavaScript ES6+ (destructuring, async/await, modules, array methods, closures)
- React with hooks or Vue/Svelte (component model, state, lifecycle)
- TypeScript (both client and server — not just "has tsconfig.json")
- Node.js fundamentals (event loop understanding, CommonJS vs ESM, async patterns)
- REST API design and implementation (Express, Fastify, or Hono — proper routing, middleware, status codes)
- Relational database integration (PostgreSQL or MySQL, parameterized queries, schema design)
- Authentication and authorization (JWT or session cookies, password hashing with bcrypt)
- Input validation and basic security (SQL injection prevention, CORS, environment variables for secrets)
- Git workflow (feature branches, pull requests, meaningful commit messages)

## Bonus Concepts

- ORM usage (Prisma, Drizzle — schema definition, migrations)
- Docker and docker-compose for local development
- API testing (supertest, Vitest, or Jest for integration tests)
- React Query or TanStack Query (distinguishing server state from client state)
- tRPC or GraphQL (type-safe API layer)
- Caching patterns (Redis, HTTP cache headers)
- WebSockets or server-sent events
- CI/CD pipelines (GitHub Actions, automated testing and deployment)
- Error monitoring (Sentry or equivalent — both client and server error capture)
- Environment configuration management (dotenv, config validation, secrets not in source)

## Complexity Threshold

minimum_complexity_score: 45
