# Junior Frontend Engineer

An upper-junior frontend engineer who has moved beyond tutorials. They build real, multi-page
applications using framework idioms correctly — not just "knows React" but understands hooks,
composition patterns, and how state flows through an app.

## Required Concepts

- HTML and CSS (flexbox, grid, responsive layout, semantic elements)
- JavaScript ES6+ (destructuring, async/await, modules, array methods, closures)
- React with hooks (useState, useEffect, useCallback, useMemo, custom hooks)
- Component composition (lifting state, prop drilling awareness, context for cross-cutting concerns)
- State management beyond useState (Zustand, Redux Toolkit, Jotai, or Recoil)
- REST API integration (fetch or axios with loading, error, and empty states handled)
- TypeScript in React context (props typing, event types, generics for API responses)
- Git workflow (feature branches, pull requests, meaningful commit messages)
- Modern build tooling (Vite, or equivalent — not exclusively Create React App)

## Bonus Concepts

- React Query or TanStack Query (server state vs client state distinction)
- React Testing Library and Vitest or Jest
- Utility-first CSS or CSS-in-JS (Tailwind CSS, styled-components, CSS Modules)
- Form libraries with validation (react-hook-form, Zod schema validation)
- Accessibility fundamentals (ARIA roles, semantic HTML, keyboard navigation)
- Deployment and CI/CD (Vercel, Netlify, GitHub Actions)
- Performance patterns (code splitting, lazy loading, memoization trade-offs)
- Vue or Svelte as an alternative to React
