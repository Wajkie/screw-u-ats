# Mid C# / .NET Engineer

A mid-level C# engineer with 2–4 years of production experience on the .NET stack. They go
beyond basic CRUD controllers — they own database migrations, background processing, structured
observability, and integration test coverage. Architectural patterns like CQRS/MediatR and
containerized deployments are routine, not aspirational.

## Required Concepts

- efcore migrations query | EF Core migrations and query optimization (eager loading, compiled queries, explain plans)
- hangfire hosted | Background jobs (Hangfire, IHostedService, or Quartz.NET)
- mediatr cqrs | MediatR and CQRS pattern (command/query separation, pipeline behaviors)
- webapplication integration | Integration testing with WebApplicationFactory (full request pipeline, real DB)
- docker dotnet | Docker and containerization for .NET (multi-stage Dockerfile, docker-compose, health checks)
- serilog logging | Structured logging with Serilog (sinks, enrichers, correlation IDs, log levels)
- versioning aspnet | API versioning in ASP.NET Core (URL segments, header negotiation, deprecation)

## Bonus Concepts

- aspnet | Minimal API vs controller-based trade-offs and migration patterns
- signalr | SignalR for real-time communication (hub lifecycle, groups, connection management)
- dotnet | Clean Architecture or Vertical Slice Architecture in .NET
- polly | Resilience and transient fault handling with Polly (retry, circuit breaker, bulkhead)
- csharp | Source generators or Roslyn analyzers (compile-time code generation)
- dotnet | OpenTelemetry instrumentation for .NET (traces, metrics, logs pipeline)
- kubernetes dotnet | Kubernetes deployment for .NET workloads (liveness/readiness probes, ConfigMaps)
- efcore | Multi-tenancy strategies in EF Core (row-level filtering, schema-per-tenant)
